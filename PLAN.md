# ДомГость — План реализации сайта бронирования жилья

> **Для исполнителя (Sonnet):** Это полная спецификация. Реализуй её точно, файл за файлом, в порядке из раздела «Порядок работ». Никакого бэкенда, никакого build-шага — чистые HTML/CSS/JS. Сайт должен открываться двойным кликом по `index.html` и одинаково работать после деплоя на Netlify.

---

## 0. Решения проекта (зафиксированы, не менять)

| Параметр | Значение |
|---|---|
| Название бренда | **ДомГость** |
| Слоган | «Жильё для отдыха напрямую от хозяев» |
| Стек | Чистые HTML + CSS + Vanilla JS, **без сборки, без фреймворков, без npm** |
| Язык контента | Русский |
| Бэкенд | Нет. Данные — в `assets/js/data.js`. Состояние (избранное, заявки) — `localStorage` |
| Оплата | Нет. Только заявка на бронирование (форма → сохранение в localStorage + сообщение об успехе) |
| Хостинг | Netlify (статика) |

Прообраз — forento.ru (поиск жилья, карточки объектов, страница объекта, заявка хозяину, кабинет «сдать жильё»). **Дизайн и название должны полностью отличаться** от оригинала.

## 1. Фирменный стиль

### Цвета (CSS-переменные)
```css
:root{
  --navy:#1b2a4a;        /* основной тёмный, шапка, заголовки */
  --navy-2:#2e4374;      /* акцент-синий, ховеры */
  --coral:#ff6b6b;       /* кнопки, акценты, цена */
  --coral-dark:#e85555;  /* ховер кнопок */
  --bg:#f7f9fc;          /* фон страницы */
  --surface:#ffffff;     /* карточки */
  --text:#1f2430;        /* основной текст */
  --muted:#6b7280;       /* второстепенный текст */
  --line:#e6eaf0;        /* границы/разделители */
  --star:#ffb400;        /* рейтинг */
  --success:#2bb673;     /* успех */
  --shadow:0 6px 24px rgba(27,42,74,.08);
  --shadow-lg:0 12px 40px rgba(27,42,74,.14);
  --radius:16px;
  --radius-sm:10px;
  --container:1200px;
}
```

### Типографика
- Подключить Google Fonts через `<link>`: **Manrope** (текст, 400/500/700/800) и **Unbounded** (заголовки/логотип, 600/700).
- Заголовок логотипа: «Дом» цветом `--navy`, «Гость» цветом `--coral`, шрифт Unbounded. Слева — иконка домика (inline SVG).
- Базовый размер 16px, líne-height 1.6, заголовки Unbounded с межбуквенным -0.01em.

### Общие компоненты (стилизовать один раз в styles.css)
- `.btn` (coral, заливка), `.btn--outline`, `.btn--ghost`, `.btn--lg`
- `.card` (белая, radius, shadow, hover: translateY(-4px) + shadow-lg)
- `.badge`, `.chip` (фильтры/категории), `.rating` (звезда + число)
- `.container` (max-width var(--container), padding 0 20px, margin auto)
- `.section` (padding 72px 0), `.section__title`, `.section__subtitle`
- Тосты `.toast` (фикс. внизу справа, success-зелёный), модалка `.modal` + `.modal__overlay`
- Адаптив: брейкпоинты 1024 / 768 / 560. На мобильном — бургер-меню, фильтры в выдвижной панели, сетки в 1 колонку.

## 3. Модель данных (`assets/js/data.js`)

Экспортировать глобально: `window.DATA = { listings:[...], cities:[...], reviews:[...] }`.

### Объект жилья (listing)
```js
{
  id: "p1",
  title: "Уютные апартаменты у моря",
  type: "Квартира",          // один из: "Отель" | "Квартира" | "Гостевой дом" | "Коттедж"
  category: "Море",          // "Море" | "Город" | "Горы" | "Природа"
  city: "Сочи",
  region: "Краснодарский край",
  address: "ул. Приморская, 12",
  price: 4500,               // ₽ за ночь
  rating: 4.8,
  reviewsCount: 124,
  guests: 4,                 // макс. гостей
  bedrooms: 2,
  beds: 3,
  baths: 1,
  area: 56,                  // м²
  images: [ /* 4–6 URL, см. §6 */ ],
  amenities: ["Wi-Fi","Кухня","Кондиционер","Парковка","Стиральная машина","Балкон"],
  description: "2–3 абзаца текста на русском…",
  host: { name:"Анна", since:2019, superhost:true },
  badges: ["Топ-выбор"],     // опционально: "Хит", "Новинка", "Без комиссии"
  reviewIds: ["r1","r3"]     // ссылки на reviews
}
```

### Требования к данным
- **Минимум 18 объектов**, равномерно по категориям (Море/Город/Горы/Природа) и типам.
- Города (реальные для РФ/Абхазии, как у прообраза): Сочи, Ялта (Крым), Гагра (Абхазия), Дербент (Дагестан), Москва, Санкт-Петербург, Казань, Красная Поляна, Эльбрус, Домбай, Калининград, Суздаль.
- Цены 1500–25000 ₽, рейтинги 4.3–5.0, у части объектов badges.
- `cities`: массив `{name, region, category, image, count}` — 8 карточек популярных направлений для главной.
- `reviews`: `{id, author, avatarLetter, city, rating, date:"Март 2026", text}` — минимум 8 штук.

## 4. Страницы — детальная спецификация

Во всех страницах: единая `<header>` (рендерит `ui.js`) и `<footer>`. Шапка sticky, при скролле лёгкая тень.

### Шапка (на всех страницах)
- Логотип ДомГость (ссылка на index).
- Меню: Главная · Каталог · Сдать жильё · О сервисе.
- Справа: ссылка «Избранное ♥» со счётчиком-бейджем (из localStorage), кнопка-coral «Сдать жильё».
- Мобильный: бургер → выезжающее меню.

### Футер (на всех страницах)
4 колонки: О бренде+слоган+соцыконки (SVG) · Направления (ссылки на catalog с фильтром) · Сервис (ссылки) · Контакты (телефон, email, город). Низ: «© 2026 ДомГость. Все права защищены.» + «Сервис создан в демонстрационных целях.»

### 4.1 `index.html` — Главная
1. **Hero**: фоновое фото (overlay navy-градиент), H1 «Найдите идеальное жильё для отдыха», подзаголовок-слоган. **Поисковая панель** (белая карточка): поле «Куда» (select/датализт городов), «Заезд» и «Выезд» (`<input type="date">`), «Гости» (счётчик −/+), кнопка-coral «Найти». Submit → переход `catalog.html?city=...&checkin=...&checkout=...&guests=...`.
2. **Категории**: 4 крупные плитки с фото (Море, Город, Горы, Природа) → `catalog.html?category=Море` и т.д.
3. **Популярные направления**: сетка карточек городов (из `DATA.cities`) с фото и «N объектов» → catalog с фильтром по городу.
4. **Лучшие предложения**: сетка карточек объектов (топ по рейтингу, 8 шт.), компонент карточки из `ui.js`, кнопка «Смотреть все» → catalog.
5. **Как это работает**: 3 шага с иконками (1. Найдите жильё · 2. Отправьте заявку хозяину · 3. Отдыхайте без комиссии).
6. **Статистика**: 4 счётчика с анимацией при появлении (100 000+ объектов · 1.5 млн гостей · 0% комиссия · 14 лет на рынке).
7. **Отзывы**: слайдер/сетка из `DATA.reviews` (3 видимых, листание).
8. **CTA-баннер**: «Сдаёте жильё? Размещайте объявления бесплатно» → owners.html.
9. Футер.

### 4.2 `catalog.html` — Каталог
- Хлебные крошки + заголовок с городом/категорией из query.
- **Сайдбар фильтров** (слева, на мобильном — кнопка «Фильтры» открывает панель):
  - Тип жилья (чекбоксы), Категория (чипы), Цена (два поля «от/до» или ползунок), Гости (счётчик), Удобства (чекбоксы), кнопка «Сбросить».
- Верхняя панель: число результатов + сортировка (по популярности/цене ↑/цене ↓/рейтингу).
- Сетка карточек (3 кол. desktop / 2 / 1). Карточка из `ui.js`.
- Фильтрация **мгновенная** на JS (без перезагрузки), результаты обновляются. Параметры пишутся в URL (history.replaceState) — чтобы ссылка сохраняла фильтр.
- Пустое состояние: иллюстрация + «Ничего не найдено, измените фильтры».

### 4.3 `property.html` — Страница объекта
Читает `?id=` → находит объект в `DATA.listings`. Если нет — показать сообщение/редирект.
- **Галерея**: крупное фото + сетка превью; клик → лайтбокс (листание ←/→, закрытие Esc/клик по фону).
- Заголовок, тип, рейтинг+отзывы, локация (город, регион, адрес).
- Кнопки «♥ В избранное» (toggle) и «Поделиться» (copy link → тост).
- **Левая колонка**: характеристики (гостей/спален/кроватей/санузлов/площадь иконками), описание, сетка удобств с иконками, блок о хозяине (аватар-буква, имя, «Хозяин с 2019», бейдж «Суперхозяин»), карта — статичный блок-плейсхолдер с адресом (можно `<iframe>` OpenStreetMap по координатам ИЛИ просто стилизованный блок с пометкой адреса; без API-ключей).
- **Правая колонка — sticky карточка бронирования**: цена/ночь, поля Заезд/Выезд/Гости, авторасчёт: `ночей × цена` + строка итога. Кнопка-coral «Забронировать» → **модалка заявки**.
- **Модалка заявки**: поля Имя*, Телефон*, Email, даты/гости (предзаполнены), Сообщение. Валидация (имя ≥2, телефон по маске/непустой). Submit → сохранить заявку в `localStorage` (ключ `dg_bookings`), закрыть модалку, показать success-экран «Заявка отправлена хозяину! Он свяжется с вами.» Никаких реальных запросов.
- **Похожие объекты**: 3–4 карточки той же категории/города.

### 4.4 `favorites.html` — Избранное
- Читает id из `localStorage` (`dg_favorites`), рендерит карточки. Кнопка-сердце удаляет из избранного и из сетки.
- Пустое состояние: «В избранном пока пусто» + кнопка «Перейти в каталог».

### 4.5 `owners.html` — Сдать жильё
- Hero с CTA. Блок преимуществ (Бесплатно · Прямые заявки с контактами гостя · Вы сами определяете условия и предоплату · 0% комиссии) — как у прообраза.
- «Как разместить» 3–4 шага.
- **Форма размещения**: Тип жилья, Город, Заголовок, Цена/ночь, Кол-во гостей, Описание, Имя, Телефон, Email, согласие (checkbox). Валидация. Submit → сохранить в `localStorage` (`dg_listings_submitted`) + success-тост «Заявка на размещение принята! Модератор свяжется с вами.»
- FAQ-аккордеон (4–6 вопросов).

### 4.6 `about.html` — О сервисе
- О бренде, миссия (напрямую от хозяев, без комиссии, безопасно).
- «Как это работает» (для гостей и для хозяев).
- Преимущества/гарантии (проверенные хозяева, реальные отзывы).
- FAQ-аккордеон. Контакты.

### 4.7 `404.html`
Простая страница с логотипом и ссылкой на главную.

## 5. JavaScript — функциональность

### `ui.js` (общий, грузится на всех страницах)
- `renderHeader()` / `renderFooter()` — вставляют разметку, подсвечивают активный пункт меню, вешают бургер.
- Избранное: `getFavorites()`, `toggleFavorite(id)`, `isFavorite(id)`, `updateFavCount()` — через `localStorage` ключ `dg_favorites` (массив id). Сердечки на карточках синхронизируются.
- `renderListingCard(listing)` → строка HTML карточки: фото (первое из images, `loading="lazy"`, onerror-фолбэк), бейджи, сердце-toggle, тип·город, заголовок (ссылка `property.html?id=`), рейтинг, цена «от N ₽ / ночь».
- Утилиты: `formatPrice(n)` → «4 500 ₽», `nightsBetween(a,b)`, `qs()`/`getParam()`, `showToast(msg, type)`, `escapeHtml()`.
- Анимация появления секций (IntersectionObserver, класс `.in-view`).

### Страничные скрипты
- `home.js`: обработчик формы поиска → сборка URL; рендер категорий/городов/топ-объектов/отзывов; анимация счётчиков статистики.
- `catalog.js`: чтение query → начальные фильтры; функция `applyFilters()` фильтрует `DATA.listings` по типу/категории/цене/гостям/удобствам, сортирует, ререндерит сетку и счётчик; пишет состояние в URL; кнопка сброса; на мобильном открытие панели фильтров.
- `property.js`: рендер объекта; галерея+лайтбокс; калькулятор (пересчёт при смене дат/гостей); модалка+валидация+сохранение заявки+success; рендер похожих; кнопки избранное/поделиться.
- `favorites.js`: рендер из localStorage, обработка удаления, пустое состояние.
- `owners.js`: валидация формы, сохранение, тост успеха, сброс формы.

Всё на чистом JS (без зависимостей). Без `eval`, без внешних библиотек. Иконки — inline SVG (набор: дом, сердце, звезда, wifi, парковка, кухня, кондиционер, гость, кровать, ванна, площадь, локация, телефон, почта, стрелки, бургер, крест, соцсети).

## 7. Netlify деплой

`netlify.toml`:
```toml
[build]
  publish = "."
[[redirects]]
  from = "/*"
  to = "/404.html"
  status = 404
```
- Поскольку сборки нет — деплой простым перетаскиванием папки в Netlify Drop **или** подключением Git-репозитория (publish dir = корень, build command пустой).
- Все пути к ассетам — **относительные** (`assets/css/...`), без ведущего слэша, чтобы работало и локально, и на Netlify.
- `robots.txt`: allow all. Базовые мета-теги (title, description, og:title, og:image, viewport, charset) на каждой странице.

`README.md` (на русском) — короткая инструкция: как открыть локально и 2 способа деплоя на Netlify.
